# 태블릿 현장 필기 인식

## 목표

현장에서는 키보드 입력을 요구하지 않는다. 펜으로 쓴 원본 획을 먼저 안전하게 저장하고, 태블릿이 읽은 자동 전사문과 사람이 확인한 최종 전사문을 별도로 보존한다. 데스크탑 보고서 작성에서는 최종 전사문을 우선 사용하되 원필기를 항상 대조할 수 있어야 한다.

## 저장 계약

- `penMemoStrokes`: 펜 획 원본. 인식 성공 여부와 무관하게 보존한다.
- `penMemoAutoTranscript`: 태블릿 인식기가 처음 읽은 문자열.
- `penMemoReviewedTranscript`: 조사자가 원필기와 대조해 확정한 문자열.
- `penMemoTranscriptionStatus`: `pending`, `autoTranscribed`, `reviewed`, `failed`.
- `penMemoRecognitionEngine`: 인식 결과를 만든 엔진과 버전.
- `penMemoRecognitionConfidence`: 엔진이 문장 신뢰도를 제공할 때만 기록한다.

## 현장 흐름

1. 유구 카드에서 `메모 +`를 누른다.
2. 전체화면 캔버스에 펜으로 기록한다.
3. 원필기를 저장한 뒤 `이 필기 읽기`를 누른다.
4. 자동 전사 후보를 확인하고 필요한 부분만 고친다.
5. `검토한 글자 확정`으로 보고서용 텍스트를 남긴다.

한국어 모델은 최초 한 번만 내려받는다. 이후 인식은 태블릿 안에서 오프라인으로 실행하며 펜 획을 외부 서버로 전송하지 않는다.

## 삼성 태블릿 기본 기능과 역할 분담

현장 태블릿에는 삼성 필기 서비스, 삼성 키보드, 삼성 노트가 기본 설치되어 있다. 삼성 키보드의 S펜 텍스트 입력은 자동 전사문을 고치는 입력란에서 그대로 활용한다. 삼성 노트의 필기 변환은 별도 앱 안에서는 우수하지만, iDAI.field 기록의 유구 관계·원본 획·자동 전사문·검토 전사문을 한 저장 단위로 돌려주는 공개 연동 API가 아니므로 주 기록 화면으로 대체하지 않는다.

앱은 삼성 노트의 편집 기능을 복제하지 않는다. 유구에 귀속되는 원필기 보존, 보조 자동 인식, 보고서 이관만 담당하고 최종 교정은 태블릿 기본 키보드도 자유롭게 사용할 수 있는 일반 텍스트 입력란으로 제공한다.

실사용 엔진은 현재 앱이 이미 저장하는 획 좌표와 순서를 직접 받아 문장으로 변환할 수 있어야 한다. Android 구현은 앱의 Kotlin 1.9 빌드와 호환되는 ML Kit Digital Ink Recognition 18.1의 한국어 `ko` 모델을 사용한다. 공개 프로젝트의 코드는 복사하지 않으며 향후 현장 필기 표본으로 자체 평가할 때 비교 기준으로만 사용한다.

## 인식 품질 원칙

- 자동 전사문은 원문으로 간주하지 않는다. 보고서에는 검토 전사문을 우선한다.
- 고유명사, 유구 번호, 수치, 방향은 반드시 원필기와 대조한다.
- 실패해도 원필기와 `failed` 상태를 남겨 데스크탑 검토 대기열에 보낸다.
- 같은 조사자의 교정 결과를 향후 사전 문맥과 평가 데이터로 사용할 수 있도록 자동 전사문을 덮어쓰지 않는다.
